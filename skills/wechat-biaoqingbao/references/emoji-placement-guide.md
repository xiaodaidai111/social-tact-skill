# 微信表情包插入指南

## 总原则

表情包的作用是补充语气，不是替代内容。

```text
先把话说清楚，再用表情包调温度。
```

## 常见插入位置

### 1. 句末

最稳妥。适合感谢、确认、轻提醒、夸奖。

```text
辛苦你啦，这版已经很清楚了。[表情包: 点头/感谢]
```

### 2. 轻松转折后

适合朋友、同学、熟同事。

```text
我刚才还没反应过来，你这个思路确实妙。[表情包: 捂脸笑/鼓掌]
```

### 3. 请求之后

先说清楚请求，再用表情包降低压力。

```text
方便的话，今天下午帮我看一眼这个版本就行，不急着长篇反馈。[表情包: 抱拳/拜托]
```

### 4. 边界之后

拒绝时先把边界说清楚，表情包放最后缓和。

```text
这次我可能没法一起去了，最近时间确实排不开。下次提前约我一定去。[表情包: 抱歉/挠头]
```

### 5. 不建议插在开头

开头就放表情包容易显得不认真，尤其是请求、道歉、催进度。

不建议：

```text
[表情包: 捂脸] 你能不能帮我看看这个？
```

更好：

```text
想麻烦你帮我看一眼这个版本，有空的时候给我两个建议就行。[表情包: 抱拳]
```

## 按场景推荐

| 场景 | 推荐表情包类型 | 位置 | 避免 |
| --- | --- | --- | --- |
| 夸领导/老师 | 点头、感谢、轻微鼓掌 | 句末，可不用 | 夸张膜拜、撒娇、暧昧 |
| 夸朋友 | 鼓掌、笑哭、可爱小动物 | 句末或玩笑后 | 过度阴阳怪气 |
| 感谢别人 | 感谢、鞠躬、抱拳 | 句末 | 大笑、敷衍 OK |
| 道歉 | 抱歉、挠头、低头 | 道歉和补救后 | 搞笑、卖萌掩盖问题 |
| 拒绝 | 挠头、抱歉、微笑 | 边界说清后 | 先发表情再拒绝 |
| 求人办事 | 抱拳、拜托、感谢 | 请求说清后 | 连发表情施压 |
| 催进度 | 点头、微笑、辛苦了 | 句末 | 生气、催促、阴阳怪气 |
| 群聊接话 | 笑哭、鼓掌、收到 | 短句后 | 刷屏 |
| 朋友圈评论 | 鼓掌、星星眼、笑脸 | 句末 | 太私密、太长 |

## 按关系推荐

### 领导 / 老师 / 客户

少用，最多一个。

推荐：

- 点头
- 感谢
- 抱拳
- 轻微鼓掌

不推荐：

- 撒娇
- 亲亲
- 夸张膜拜
- 大量可爱表情

### 同事 / 同学

可以更自然，但别影响正事。

推荐：

- 笑哭
- OK
- 鼓掌
- 抱拳

### 朋友

可以更活泼。

推荐：

- 可爱小动物
- 捂脸
- 哈哈大笑
- 加油
- 贴贴类表情，但要看关系

## 数量建议

- 正式关系：0-1 个。
- 普通同事/同学：1 个。
- 熟朋友：1-2 个。
- 群聊：少发，避免刷屏。
